# 0단계: TTS/음성 클로닝 머신러닝 개념 이해

## 학습 목표

이 문서를 읽고 나면 다음을 이해할 수 있습니다:

- 음성 클로닝에서 머신러닝 모델이 어떤 역할을 하는지 설명할 수 있다
- XTTS v2 같은 모델이 어떤 과정을 통해 만들어지는지 큰 흐름을 설명할 수 있다
- 추론(Inference), 클로닝(Zero-shot), 파인튜닝(Fine-tuning)의 차이를 구분할 수 있다
- 왜 4단계(파인튜닝)가 필요한지 품질 관점에서 판단할 수 있다

**예상 소요 시간:** 15~25분  
**난이도:** ★★☆☆☆ (개념 입문)

## 사전 지식

- Python/CLI 기초
- [99-glossary.md](./99-glossary.md)의 주요 용어

## 한눈에 보는 전체 흐름

```text
[대규모 음성+텍스트 데이터]
          ↓ (사전학습)
    [XTTS v2 기본 모델]
          ↓ (3단계: 추론/클로닝)
 [레퍼런스 음성 + 텍스트 입력]
          ↓
      [합성 음성 출력]
          ↓ (4단계: 파인튜닝)
 [내 데이터에 더 맞는 커스텀 모델]
```

핵심은 다음입니다:

1. **모델 제작(사전학습)**은 모델 개발팀이 미리 수행
2. 우리는 주로 **모델 사용(추론)**을 먼저 하고
3. 필요하면 **추가 학습(파인튜닝)**으로 품질을 끌어올림

## XTTS v2는 어떻게 만들어지나?

아래는 실무에서 쓰는 일반적인 TTS 모델 개발 과정을 학습용으로 단순화한 버전입니다.

### 1) 데이터 준비

- 다양한 화자/언어의 `음성 + 전사 텍스트` 쌍을 대량 확보
- 샘플레이트, 음량, 라벨 품질을 정리
- 잘못된 전사, 잡음 과다 데이터 제거

### 2) 전처리 및 정렬

- 문장 단위로 음성을 분할
- 텍스트와 음성 타이밍 정합성 확인
- 모델 입력 포맷으로 변환

### 3) 사전학습 (Pretraining)

- 모델이 "텍스트를 음성 특징으로 바꾸는 규칙"을 학습
- 동시에 "화자 특성(음색/말투)을 조건으로 반영"하는 방법을 학습
- 다국어 학습으로 언어별 발음/리듬 패턴을 학습

### 4) 제로샷 클로닝 성능 확보

- 짧은 레퍼런스 음성만으로 화자 특성을 추정하도록 학습
- 결과적으로 사용자는 별도 학습 없이도 클로닝 시도 가능

### 5) 배포 및 추론

- 학습된 가중치를 배포
- 사용자는 `speaker_wav + text + language`로 음성 생성

## 클로닝/추론/파인튜닝 차이

| 구분 | 무엇을 하나? | 데이터 필요량 | 시간/비용 | 결과 특성 |
|------|---------------|---------------|-----------|-----------|
| 추론 (Inference) | 이미 학습된 모델로 결과 생성 | 매우 적음 | 낮음 | 빠른 시작, 품질은 기본 모델 의존 |
| 제로샷 클로닝 | 짧은 레퍼런스 음성으로 화자 조건 반영 | 낮음 (6초+) | 낮음~중간 | "비슷한 목소리"를 빠르게 생성 |
| 파인튜닝 | 내 데이터로 모델 추가 학습 | 높음 | 높음 | 음색/발음/일관성 품질 향상 |

## 왜 파인튜닝이 필요한가?

3단계 클로닝이 성공해도 다음 문제가 남을 수 있습니다:

- 문장 길이가 길어질수록 억양이 불안정
- 특정 발음(받침, 숫자, 고유명사) 품질 저하
- 문장마다 화자 느낌이 조금씩 달라짐

파인튜닝의 목적은 위 문제를 줄여 **실사용 품질(일관성/자연스러움/재현성)**을 높이는 것입니다.

## 실습: 개념을 확인하는 빠른 명령

아래 명령은 "모델을 어떻게 쓰는지" 감을 잡기 위한 최소 확인 절차입니다.

### 1) XTTS 모델 존재 확인

```bash
cd ~/workspaces/ai-voice-lab
source .venv/bin/activate
tts --list_models | grep xtts_v2
```

### 2) PyTorch 디바이스 확인 (CPU/CUDA/MPS)

```bash
python -c "import torch; print('cuda:', torch.cuda.is_available()); print('mps:', torch.backends.mps.is_available())"
```

### 3) 클로닝 실행 예시 (추론 단계)

```bash
python scripts/clone_voice.py \
  --text "안녕하세요. 개념 확인용 합성 테스트입니다." \
  --speaker_wav samples/voice_ref_01_24k.wav \
  --use-gpu
```

> 이 명령은 새 모델을 학습하는 것이 아니라, 이미 학습된 XTTS v2를 사용하는 **추론**입니다.

## 자주 생기는 오해

### Q1. 클로닝을 실행하면 내 모델이 학습되나요?

- 보통 아닙니다.
- 3단계는 대부분 추론/제로샷 클로닝으로, 모델 가중치 업데이트가 없습니다.

### Q2. 레퍼런스 음성이 6초면 항상 고품질이 나오나요?

- 아닙니다.
- 6초는 "작동 가능한 최소 수준"이고, 품질은 보통 15초+ 깨끗한 샘플에서 더 안정적입니다.

### Q3. GPU를 쓰면 품질이 무조건 더 좋아지나요?

- 주로 속도 이점입니다.
- 품질 개선은 데이터 품질과 파인튜닝 전략 영향이 더 큽니다.

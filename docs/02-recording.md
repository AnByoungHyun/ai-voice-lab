# 2단계: 음성 샘플 녹음 가이드

## 학습 목표

이 단계를 마치면 다음을 할 수 있습니다:

- XTTS v2 클로닝용 음성 샘플을 올바른 품질로 녹음할 수 있다
- 녹음 파일을 권장 포맷(`wav`, 24kHz, mono)으로 정리할 수 있다
- 잡음, 볼륨, 발화 속도 등 품질 체크 기준을 적용할 수 있다
- 다음 단계(클로닝)에서 바로 사용할 입력 파일을 준비할 수 있다

**예상 소요 시간:** 20~40분  
**난이도:** ★★☆☆☆ (초급)

## 사전 지식

- [1단계: 환경 설정](./01-setup.md) 완료
- 기본 오디오 파일 개념(샘플레이트, mono/stereo) 이해

## 준비물

- 조용한 공간(에어컨/선풍기/키보드 소음 최소화)
- 마이크(내장 마이크도 가능, 외장 USB 마이크 권장)
- 녹음 앱(QuickTime, Voice Memos, Audacity 중 하나)

## 권장 녹음 스펙

| 항목 | 권장값 | 비고 |
|------|--------|------|
| 파일 형식 | WAV | 무손실 포맷 권장 |
| 샘플레이트 | 24,000 Hz | XTTS v2 기본 입력에 맞추기 좋음 |
| 채널 | Mono (1ch) | 스테레오보다 처리 단순 |
| 길이 | 15~60초 | 최소 6초 가능하나 15초 이상 권장 |
| 파일 수 | 1~3개 | 처음에는 1개로 시작 |

## 녹음 스크립트 예시

아래 문장 중 2~3개를 자연스럽게 읽어 녹음합니다.

1. "안녕하세요. AI Voice Lab 음성 클로닝 실습을 진행합니다."
2. "오늘은 제 목소리를 기반으로 텍스트를 음성으로 변환해보겠습니다."
3. "녹음은 조용한 환경에서 또렷하고 일정한 톤으로 진행합니다."

> 너무 빠르게 읽거나 감정을 과하게 넣기보다, 평소 말투에 가깝게 읽는 것이 좋습니다.

## 단계별 진행

### 1) 녹음

- 입과 마이크 거리를 15~20cm 정도로 유지
- 녹음 전 2초 정도 무음 구간 확보
- 중간 실수가 있으면 처음부터 다시 짧게 녹음

### 2) 파일 저장

- 프로젝트 경로의 `samples/` 디렉토리에 저장
- 파일명은 일관되게 관리 (예: `voice_ref_01.wav`)

```bash
cd ~/workspaces/ai-voice-lab
mkdir -p samples
```

### 3) 오디오 포맷 정리 (선택)

`ffmpeg`가 있다면 아래 명령으로 포맷을 통일합니다.

```bash
ffmpeg -i samples/voice_ref_01.wav -ac 1 -ar 24000 samples/voice_ref_01_24k.wav
```

### 4) 길이/샘플레이트 확인

```bash
ffprobe -hide_banner samples/voice_ref_01_24k.wav
```

출력에서 다음 항목을 확인하세요:
- `Audio: pcm_s16le`
- `24000 Hz`
- `mono`

## 품질 체크리스트

- [ ] 배경 소음(팬 소리, 키보드 소리, 반향)이 거의 없다
- [ ] 말의 시작/끝이 잘리지 않았다
- [ ] 음량이 너무 작거나 과하게 크지 않다 (클리핑 없음)
- [ ] 발음이 또렷하고 속도가 일정하다
- [ ] 15초 이상 유효 발화가 포함되어 있다

## 자주 발생하는 문제와 해결

### 잡음이 많이 섞이는 경우

- 창문 닫기, 냉난방기 일시 정지, 키보드 타건 중지
- 마이크 입력 감도를 낮추고 입과 거리를 일정하게 유지

### 소리가 너무 작거나 큰 경우

- macOS 입력 볼륨을 중간(약 60~70%)으로 조정
- 파형이 계속 상단/하단에 붙으면(클리핑) 다시 녹음

### 파일 포맷이 맞지 않는 경우

- `ffmpeg` 명령으로 `mono + 24kHz + wav`로 재변환

## 산출물

- `samples/voice_ref_01_24k.wav` (권장)
- 또는 동일 스펙의 사용자 음성 파일 1개 이상

> `samples/` 디렉토리는 개인 음성 데이터이므로 절대 커밋하지 않습니다.
